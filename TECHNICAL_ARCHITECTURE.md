## Privacy And Security Principles

Veridul is designed with privacy as a core principle.

### Data Minimization

Veridul should collect only the information necessary to assess communication risk.

### On-Device Processing

Where technically possible, communication analysis should occur directly on the user's device.

### Encryption

Any data transmitted between the device and Veridul services must be encrypted.

### Storage

Call audio should not be stored by default.

Temporary processing data should be deleted after analysis whenever possible.

### User Control

Users should have clear control over permissions and privacy settings.

### Security Objective

Veridul exists to protect communications, not to collect communications.

Privacy and trust are fundamental product requirements.
